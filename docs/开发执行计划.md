# DK日常开销 —— 开发执行计划

## 开发策略

本项目的开发策略为**分阶段渐进式开发**，每个阶段完成一个独立可用的功能子集。每个阶段完成后，用户可实际使用并反馈，再进入下一阶段。

---

## 阶段总览

| 阶段 | 名称 | 状态 |
|------|------|------|
| 一 | 项目骨架 + 规范文档 | ✅ 已完成 |
| 二 | 核心记账功能（全部功能开发） | ✅ 已完成 |
| 三 | 部署 + iPhone 真机测试 | ⬅️ 当前阶段 |
| 四 | 打磨优化（按需） | ⏳ 待开始 |

## 阶段性提交

每个阶段完成后，需：
1. 更新 `docs/开发执行计划.md`（标记已完成项）
2. 编写当日 `devlog/` 日志
3. 向用户汇报完成情况和下一阶段计划
4. 等待用户确认后再进入下一阶段
